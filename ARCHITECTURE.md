# パッティング分析アプリ - システム構成・コスト・料金設計

## 1. システム全体像

```
┌─────────────────────────────────────────────────┐
│  ユーザー（スマホ/PC ブラウザ）                      │
└──────────────────┬──────────────────────────────┘
                   │ HTTPS
┌──────────────────▼──────────────────────────────┐
│  Vercel（フロントエンド + APIサーバー）              │
│  ・Expo Web（React）の静的配信                     │
│  ・tRPC API（Serverless Functions）               │
└───────┬────────────────────┬────────────────────┘
        │                    │
┌───────▼────────┐  ┌───────▼────────────────────┐
│  Supabase      │  │  Google AI Studio          │
│  ・PostgreSQL  │  │  ・Gemini 2.5 Flash        │
│  ・Auth        │  │  ・スコアカードOCR           │
│  ・Storage     │  │    (画像→テキスト)           │
└────────────────┘  └────────────────────────────┘
```

### 各サービスの役割

| サービス | 役割 | 選定理由 |
|---|---|---|
| **Vercel** | フロント配信 + API | Expo Web対応、Serverless Functions、無料枠が大きい |
| **Supabase** | DB + 認証 + ファイル保存 | PostgreSQL/Auth/Storageを一括無料提供 |
| **Google AI Studio** | スコアカード画像のOCR解析 | Gemini 2.5 Flash の無料枠が最も大きい（日250回） |

## 2. 元のアプリからの変更点（Manus依存の除去）

元のコードは「Manus」プラットフォームに依存しており、以下を置き換える。

| 元（Manus） | 新（独立運用） |
|---|---|
| Manus OAuth | Supabase Auth（メール/Google/Apple） |
| Manus SDK (LLM) | Google AI Studio Gemini API 直接呼び出し |
| Manus Storage (S3 proxy) | Supabase Storage |
| MySQL/TiDB (Manus提供) | Supabase PostgreSQL |
| Manus Runtime (iframe通信) | 削除（不要） |
| Bundle ID `space.manus.*` | 独自ドメインベース |

## 4. 規模別コスト試算

### 前提条件（1ユーザーあたり）

- 月4〜8ラウンド（週1〜2回ゴルフ）
- 1ラウンドにつきOCR 1回 + データ保存
- DB: 1ラウンド ≈ 5KB（18ホール分のパットデータ）
- ストレージ: スコアカード画像1枚 ≈ 1〜3MB

### コスト一覧

| 規模 | ユーザー数 | Vercel | Supabase | Gemini | 合計(月) | 1人あたり |
|---|---|---|---|---|---|---|
| 個人利用 | 1〜5人 | $0 | $0 | $0 | **$0** | $0 |
| 小規模 | 50人 | $0 | $0 | $0 | **$0** | $0 |
| 中規模 | 200人 | $0 | $25 | ≈$0.5 | **≈$26** | $0.13 |
| 本格 | 1,000人 | $20 | $25 | ≈$15 | **≈$61** | $0.06 |
| 大規模 | 5,000人 | $20 | ≈$40 | ≈$70 | **≈$130** | $0.03 |

### 詳細の補足

- **50人まで**: 全サービスの無料枠内に収まる（$0）
- **200人**: Supabase Free の非アクティブ停止が問題 → Pro($25)に上げる
- **1,000人**: Vercel帯域がHobby上限(100GB)に近づく → Pro($20)
- **Gemini**: 1 OCRリクエスト ≈ 5,000入力トークン + 500出力トークン ≈ $0.0028

## 6. リスクと注意点

| リスク | 対策 |
|---|---|
| Gemini無料枠の縮小（2025年12月に実績あり） | Flash-Liteへのフォールバック、有料切替の仕組みを用意 |
| Supabase Free の1週間停止 | cron等で定期アクセス or 有料ユーザーがいればPro化 |
| Vercel Serverless のコールドスタート | 画面遷移中にプリフェッチして体感速度をカバー |
| ユーザーデータのプライバシー | Gemini有料版はデータ学習に使われない。無料版は注意 |
